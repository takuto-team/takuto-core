## Non-goals

Behaviour-preserving = **pixel-preserving**.

- No visual or DOM redesign. Same `className` strings, same nested-`div` structure, same conditional layout branches. Only allowed runtime diff is extra `<>` fragments.
- No Tailwind class edits. No API/URL/shape change. `useTicketDetail`, `useTicketCountdown`, `useToast`, `useWorkflows`, `useWebSocket`, `usePolling`, `useAuth` imported as-is.
- No `useApi<T>` introduction. No new test infra, no snapshots, no new `data-testid`.
- No edit to `ui/src/api/types.ts` / `ui/src/api/client.ts` bodies. No drive-by extraction of `MarkdownPreview`, `DiffView`, `AiPromptPanel`, `Link`, icons.

## Part B — `Dashboard.tsx` (440 LOC → 1 shell + 4 hooks + 1 modals component)

### `useDashboardModals()` discriminated union

```ts
type DashboardModalState =
  | { kind: "none" }                                                                       // no modal
  | { kind: "picker" }                                                                     // <TicketPickerModal/>
  | { kind: "paste" }                                                                      // <PasteDescriptionModal/>
  | { kind: "nojira" }                                                                     // <NoJiraAlertModal/>; close() writes sessionStorage
  | { kind: "detail"; ticket: { key: string; summary: string; description?: string; url?: string; showStart: boolean } } // <TicketDetailModal/>
  | { kind: "report"; reportKey: string };                                                 // <ReportModal/>, gated on workflows[reportKey]

type Result = {
  modal: DashboardModalState;
  openPicker: () => void; openPaste: () => void; openNoJira: () => void;
  openDetail: (ticket: Extract<DashboardModalState, { kind: "detail" }>["ticket"]) => void;
  openReport: (reportKey: string) => void; close: () => void;
};
function useDashboardModals(config: ConfigResponse | null): Result;
```

Semantics: `picker` opened by `handleAddWorkflow` when `ticketingSystem !== "none"`; `paste` opened by `handleAddWorkflow` when `ticketingSystem === "none"`, submit → `none` + `fetchWorkflows()`; `nojira` auto-opens once per session when `config.ticketing_system === "none"` && `sessionStorage["noJiraAlertDismissed"] !== "1"` (L228–233), close writes the key; `detail` opened by `handleTicketSelected` or `handleShowDescription` carrying the ticket payload (picker→detail must skip `none` to avoid flicker, matching L250–251); `report` opened by `openReport(key)`, consumer gates on `workflows[reportKey] && generate_report && has_report` (L419).
