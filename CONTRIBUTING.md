# Contributing to Maestro

## Not currently accepting external contributions

Maestro is **not accepting external code contributions at this time**.

Maestro is dual-licensed (FSL + commercial). Accepting outside code
under that model requires a Contributor License Agreement (CLA) and the
tooling to track sign-offs across every PR. That infrastructure is not yet
in place, so we cannot legally accept contributions today.

This will change. Once the CLA process is set up, this file will be
replaced with the full contribution guide and the policy will be lifted.

## What you can do in the meantime

- **Report bugs and request features** via
  [GitHub Issues](https://github.com/morphet81/maestro-core/issues).
- **Ask questions or share ideas** in
  [GitHub Discussions](https://github.com/morphet81/maestro-core/discussions).
- **Report security vulnerabilities privately** — see
  [`SECURITY.md`](./SECURITY.md). Do not open public issues for vulnerabilities.

## Pull requests

Pull requests from outside the maintainer team will be politely closed.
Please open an Issue or Discussion instead — your input is welcome through
those channels.
